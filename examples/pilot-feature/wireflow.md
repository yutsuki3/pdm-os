# ワイヤーフロー: 通知設定のカスタマイズ機能（サンプル）

- 外部デザインファイル: TBD（TBD-029: 正式ツール未確定のため、本ワイヤーフローはテキストベースの草稿に留める）

## 画面一覧

| 画面ID | 画面名 | 概要 |
|---|---|---|
| S1 | アカウント設定 (トップ) | 各種設定への入口。既存画面（架空） |
| S2 | 通知設定画面 | 通知種別ごとのON/OFFを切り替える画面。本機能で追加・変更する画面 |

## フロー

```
S1(アカウント設定) --[通知設定を開く]--> S2(通知設定画面) --[種別トグルを変更]--> S2(通知設定画面)
```

トグル変更後にS2に留まる想定（別画面への遷移なし）。保存操作を伴うか、トグル変更が即時保存かは未確定（下記オープンクエスチョン参照、FR-002と同じ論点）。

## 画面ごとの注記

### S1 アカウント設定 (トップ)

- 目的: 既存のアカウント設定画面。通知設定への導線を追加する。
- 主要な要素: 「通知設定」への遷移リンク（既存要素の可能性が高いが、新規追加かどうかは未確認）。
- 分岐条件: なし。

### S2 通知設定画面

- 目的: 通知種別ごとにON/OFFを切り替える（FR-001）。
- 主要な要素: 通知種別ごとのトグルスイッチのリスト（種別の一覧・粒度はTBD、仕様書の open_questions 参照外の新規論点）。
- 分岐条件: なし（種別ごとのトグルは独立して動作する想定）。
