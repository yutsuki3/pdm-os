# ADR-0004: 事実・仮定・提案・決定を区別する文書規約を採用する

## ステータス

Accepted

## コンテキスト

PdM OSの設計ドキュメントには、ユーザーが明示的に述べた事実、確認が取れていない仮定、AIエージェントが出す設計提案、正式に承認・確定した決定が混在しうる。これらを区別せずに書くと、後から読んだ人間やAIエージェントが「これは確定した業務ルールなのか、それとも単なる提案なのか」を誤認するリスクがある。

ユーザーから「事実、仮定、提案、決定を区別する」「業務上の決定を勝手に確定しない」という作業ルールが明示された。

## 決定

ドキュメント中で紛らわしい記述には、以下のラベルを付記する。

| ラベル | 意味 |
|---|---|
| 【事実】 | ユーザーが明示的に述べた、または既存資料から直接確認できる内容 |
| 【仮定】 | 確認が取れていないが暫定的に置いている前提 |
| 【提案】 | AIエージェントが設計として提案している、ユーザー未承認の内容 |
| 【決定】 | ユーザーが承認した、またはADRとして確定した内容 |
| `TBD` | 上記いずれにも分類できない未決定事項 |

AIエージェントは、ユーザーの承認や既存ADRの裏付けなしに「決定」ラベルを使わない。未決定事項は [docs/decisions/tbd-register.md](tbd-register.md) に集約する。

## 影響

- [AGENTS.md](../../AGENTS.md) にラベル規約を明記した。
- 新規・更新される設計ドキュメント（特に `docs/architecture/`, `docs/decisions/`）でこの規約を適用する。既存の全ドキュメントへの遡及適用は行っていない（過剰な書き換えを避けるため）。

## 未確定事項

- 既存ドキュメント全体への規約の遡及適用をどこまで行うか。
