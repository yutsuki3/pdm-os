# PdM OS ビジョン

## 課題

PdM/POは、要求・要望の整理から仕様化、タスク分解、受領判断、QA依頼、リリースノート作成まで、一連の業務を横断的に担っている。これらの業務は以下の情報源にまたがっており、手作業での往復コストが大きい。

- Notion（現行仕様・意思決定）
- Confluence（過去仕様）
- Google Drive（受領原本・Google Docs）
- Jira（タスク状態）
- GitHub（実装事実）

情報が分散しているため、「今の正しい仕様は何か」「過去に同様の議論がなかったか」「実装は仕様どおりか」の確認に時間がかかり、仕様書・タスク分解・受領判断の品質と速度が安定しない。

## PdM OSとは

PdM OSは、PdM/POの以下8つの業務を支援・自動化するための仕組み（ドキュメント・スキーマ・テンプレート・プロンプト・将来的なエージェント実装）である。

1. 要求・要望から要件を整理する
2. 現行資料と過去資料を検索する
3. 仕様書を作成する
4. ワイヤーフローを作成する
5. Jiraのデザイン・実装タスクへ分解する
6. 完了成果物を受領判断する
7. QA依頼を作成する
8. リリースノートを作成する

## 対象ユーザー

- PdM (プロダクトマネージャー)
- PO (プロダクトオーナー)

エンジニア・デザイナー・QAは成果物の受け手・提供者として関わるが、一次利用者ではない。

## 対象範囲 (スコープ)

**含む**

- 要求・要望の構造化支援
- 既存資料 (Notion / Confluence / Drive) の検索・要約支援
- 仕様書・ワイヤーフローの草稿作成支援
- 仕様からJiraタスクへの分解支援
- 受領判断のためのチェック支援
- QA依頼文書の作成支援
- リリースノートの作成支援

**含まない (Non-goals)**

- PdM/POの意思決定そのものの代替（最終承認・受領判断は常に人間が行う）
- Jira / GitHub / Notion / Confluence / Google Drive 自体の代替
- 各ツールの権限管理・アカウント管理
