# プロンプト定義: run-validation-dry-run

対象エージェント: 検証を実施する機能的エージェント。特定のAIモデル・ツールに依存しない。

## 目的

実案件・正本システムへアクセスまたは書き込みをせず、架空サンプルまたは明示的に許可された成果物を使って、成果物契約と参照関係を検証する。

## 入力

- 検証対象の成果物参照
- 検証目的
- テストのために仮定する前提（ある場合）

## 手順

1. [validation-run.schema.yaml](../schemas/validation-run.schema.yaml) と [validation-run.md](../templates/validation-run.md) に従い、検証実施記録を作成する。
2. 仮定にはID、適用対象、実際の記録への参照を必ず記載する。実際の記録が `pending` / `draft` であれば、それを変更しない。
3. 各成果物の識別子・要件ID・外部参照を辿り、Schemaまたはテンプレートに不足する表現を発見した場合だけTBD候補を提示する。

## 出力

- 検証実施記録
- 成果物契約の適合状況
- Schema／テンプレート／プロンプトの改善提案
- 人間の業務判断が必要なTBD

## 禁止事項

- 仮定を実際の承認、受領、QA合否、送付、公開、状態遷移として記録すること。
- 実データ、機密情報、認証情報をサンプルや検証記録へ保存すること。
- 業務ルールを推測して確定すること。
